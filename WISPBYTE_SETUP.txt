METASTRIP ON WISPBYTE
=====================

1. Create a WispByte server using a Node.js 22 image.

2. Open Files and upload the contents of this ZIP into the server root.
   The root must contain index.js, package.json, package-lock.json, public, and lib.
   If you upload the ZIP as one archive, extract it in the file manager first.

3. Open Startup.

4. Set the startup command to:

   npm install --omit=dev && node index.js

   After the first successful start, you may change it to:

   node index.js

5. Set these environment variables under Server Configuration:

   NODE_ENV=production
   MAX_FILE_MB=40
   FILE_TTL_MINUTES=10

6. PORT SETUP

   First try using the platform's existing PORT value. The app reads process.env.PORT.

   If the website does not load:
   - Open Console and copy the port from the Address shown after the colon.
   - Add or edit the PORT environment variable with that number.
   - Restart the server.

7. Start the server. The console should show:

   MetaStrip is running on http://0.0.0.0:YOUR_PORT

8. Open the server Address. You may also create a WispByte subdomain.

RESOURCE NOTES
==============

- The app processes one upload at a time to protect the 512 MiB RAM limit.
- 40 MB is a safe starting upload limit. Lower it to 20 MB if memory usage gets high.
- FFmpeg and sharp are installed through npm. The first installation takes longer than later starts.
- Do not upload node_modules from your computer. WispByte must install Linux-compatible packages itself.
- Log in to the WispByte client panel at least once per month to keep a free server active.
